# Experiment Config Reference

All options that can appear in an experiment config JSON file.
The JSON is loaded by `ExperimentConfig.from_json()` / `from_file()` in `experiments/configs.py`.

---

## Top-level fields

| Field | Type | Required | Description |
|---|---|---|---|
| `network` | string | yes | `"sioux_falls"` or `"amsterdam"` |
| `tap_backend` | string | yes | `"fallback"`, `"aequilibrae"`, or `"surrogate"` |
| `seed` | int | yes | Master RNG seed (environment + training) |
| `run_name` | string | yes | Results written to `results/<run_name>/` |
| `instance` | string | yes | Path to instance JSON (relative to project root) |
| `training` | object | yes | See **Training options** below |
| `agent` | object | yes | See **Agent options** below |

---

## Training options (`training` object)

| Key | Type | Default | Description |
|---|---|---|---|
| `time_budget` | float | `3600.0` | Wall-clock seconds before stopping; `0` disables |
| `n_episodes` | int | `10_000_000` | Max episodes (ignored if `time_budget` stops first) |
| `eval_interval` | int | `50` | Evaluate every N training episodes |
| `update_interval` | int | `10` | Call `agent.update()` every N episodes |
| `truncation_mode` | string | `"bootstrapped"` | `"bootstrapped"` or `"none"` — whether to bootstrap the terminal value |
| `buffer_capacity` | int | `200_000` | Max transitions in replay buffer |
| `buffer_strategy` | string | `"fifo"` | `"fifo"`, `"lowest_error"`, or `"stochastic_knockout"` |
| `n_eval_episodes` | int | `10` | Episodes per evaluation pass |
| `batch_size` | int | `2048` | Transitions sampled per `update()` call |
| `n_warmstart_episodes` | int | `0` | Warm-start episodes with a heuristic before RL training |
| `warmstart` | object/null | `null` | Agent config dict for warm-start agent (same schema as `agent`) |

---

## Agent options (`agent` object)

| Key | Type | Default | Description |
|---|---|---|---|
| `agent_type` | string | — | **Required.** `"reactive"`, `"paced"`, `"dqn"`, `"actor_critic"`, `"marl"` |
| `value_fn` | string | `"xgboost"` | `"xgboost"`, `"neural"`, `"ranking"` — used by `dqn`/`actor_critic` |
| `action_gen` | string | `"local_search"` | `"local_search"`, `"sequential"`, `"bdq"` — used by `dqn`/`actor_critic` |
| `extra` | object | `{}` | Agent-type-specific options (see below) |

### `extra` options by agent type

#### `reactive`
| Key | Default | Description |
|---|---|---|
| `threshold` | `0.7` | Condition threshold above which renovation is triggered |

#### `paced`
| Key | Default | Description |
|---|---|---|
| `threshold` | `0.7` | Condition threshold triggering renovation |
| `pace_threshold` | `null` | Secondary threshold for pacing logic |

#### `dqn`
| Key | Default | Description |
|---|---|---|
| `bootstrap_mode` | `"post_decision"` | `"post_decision"`, `"next_state"`, or `"none"` |
| `finite_horizon` | `true` | Append epoch index `t` to features → shape `(5N+1,)`; `false` → `(5N,)` |

#### `actor_critic`
| Key | Default | Description |
|---|---|---|
| `bootstrap_mode` | `"post_decision"` | Same as DQN |
| `finite_horizon` | `true` | Same as DQN |
| `patience` | `20` | Policy candidate samples per decision step |
| `hidden_dims` | `[256, 256]` | Hidden layer sizes for the policy network |

## Instance JSON fields (read by `build_experiment`)

Instance files live in `instances/` and are generated by `experiments/generate_instance.py`.

### Generated per-asset arrays (length N)

| Field | Description |
|---|---|
| `d_init` | Initial condition values in [0,1] |
| `alpha0` | Baseline Gamma shape rates |
| `beta` | Gamma rate parameters |
| `mu_h` | Renovation Wiener drift (years⁻¹) |
| `sigma_h` | Renovation Wiener volatility |
| `asset_lengths_m` | Physical length per asset (metres) |
| `c_ren` | Renovation cost per asset (€) |
| `c_rep` | Repair cost per asset (€) |

### Environment scalars

| Field | Default | Description |
|---|---|---|
| `n_assets` | — | Number of assets N |
| `network` | — | Network name used during generation |
| `T` | — | Planning horizon (epochs) |
| `dt` | — | Epoch length (years) |
| `gamma` | — | Discount factor |
| `d_fail` | `1.0` | Condition threshold for failure |
| `eta_ren` | `0.05` | Capacity factor during renovation |
| `eta_load` | `0.50` | Capacity factor under load restriction |
| `delta_repair` | — | Condition improvement from repair action |
| `vot` | `10.76` | Value of time (€/vehicle-hour) |
| `traffic_cost_factor` | `1.0` | Scales raw traffic cost |
| `risk_base` | `10000.0` | €/m/year per consecutive failed epoch |
| `allow_repair` | `true` | Set `false` to permanently disable repair action |
| `allow_restrict` | `true` | Set `false` to permanently disable load-restriction action |

## Minimal example config

```json
{
  "network": "sioux_falls",
  "tap_backend": "fallback",
  "seed": 42,
  "run_name": "exp_001",
  "instance": "instances/sf_n10_s0.json",
  "training": {
    "time_budget": 3600,
    "eval_interval": 50,
    "update_interval": 10
  },
  "agent": {
    "agent_type": "dqn",
    "value_fn": "xgboost",
    "action_gen": "local_search",
    "extra": {
      "bootstrap_mode": "post_decision",
      "finite_horizon": true
    }
  }
}
```
